Atomic Design: Elementos Básicos

Botones

Solidos normal y redondeados

Por defecto se configuran los elementos: button, .at-btn, las clases at-btn-sm y at-btn-lg cambian el tamaño. La clase at-btn-round da un botón de bordes mas redondeados. Se configuran con los colores primarios (at-btn-primary) y secundarios (at-btn-secondary) respectivamente.

Para configurar un color diferente se hace uso del mixin BUTTON
.at-btn-color {
@include button( color('colección', 'color') );
}

Outline normal y redondeados

Los botones de estilo Outline se obtienen con la clase at-btn-outline

Para configurar un color diferente para botones Outline se hace uso del mixin BUTTON con los siguientes parametros
.at-btn-outline-color {
@include button( transparent, color('coleccion', 'color'), color('coleccion', 'color' ) );
border: 0.1rem solid color('coleccion', 'color' );
}

Input por Tamaño

Las etiquetas inputs cuentan con un estilo base y son los que se presentan como primera opción, luego continuan las clases input-bottomless que muestra al input con un borde inferior, input-rounded que cuenta con bordes más redondeado y outlinenone que no cuenta con bordes y tampoco outline.
La clase primary permite que el input se presente con bordes del color primario (base) y un ouline del color secundario (base), por lo contrario, la clase secondary permite que el input se presente con bordes del color secundario (base) y un ouline del color primario (base).

-------Tamaño Small-------

La clase small permite que los inputs y el font-sizes sean pequeños.

-------Tamaño Normal-------

Este tamaño se encuentra dentro del estilo base de la etiqueta input.

-------Tamaño Medium-------

La clase medium permite que los inputs y el font-sizes sean medianos.

-------Tamaño Big-------

La clase big permite que los inputs y el font-sizes sean grandes.

Radio Button

1-. Estilo base de los inputs tipo radio

2-. Para este nuevo estilo de radio button se utiliza la clase radio-primary, colocándolo en los labels para que al seleccionar se pinte del color primario (base) y tambien se debe colocar la clase group en el div que envuelve todos los inputs y label.

3-. Se debe seguir el mismo procedimiento anterior pero colocando la clase radio-secondary en los labels para que al seleccionar se pinte del color secundario (light).

ChexBox

1-. Estilo base de los inputs tipo checkbox.

2-. Para este estilo de checkbox se utiliza la clase check-secondary, colocándolo en los labels para que al seleccionar se pinte del color secundario (light) y se debe colocar la clase group-check en el div que envuelve todos los inputs y label
Si deseas se selecione del color primario, solo debes cambiar la clase check-secondary por la clase check-primary.

Input Type Submit and Type Button

-----------Estilo Base-----------

Estilo base de la etiqueta input Type Submit and Type Button.

-----------Estilo Solido-----------

El primer input posee la clase btn-primary que se presenta del color primario (base) y el segundo input posee la clase btn-secondary y se presenta del color secundario(base).

-----------Estilo Outline-----------

Las clases btn-outline-primary y btn-outline-secondary presenta bordes de color primario o secundario respectivamente.

Posicionamientos de inputs

--------input vertical--------

Las clase group-list permite que extista un posicionamiento vertical entre el label y el input.

--------input horizontal--------

Las clase group-row permite que extista un posicionamiento horizontal entre el label y el input.

--------Radio button--------

Las clase radio-list permite que extista un posicionamiento vertical entre los radios buttons.

--------checkbox--------

Las clase check-list permite que extista un posicionamiento vertical entre los checkboxs.

--------Select y Textare vertical--------

Las clase group-list permite que extista un posicionamiento vertical entre el label y el select o textarea.

--------Select y Textare horizontal--------

Las clase group-row permite que extista un posicionamiento horizontal entre el label y el select o textarea.
